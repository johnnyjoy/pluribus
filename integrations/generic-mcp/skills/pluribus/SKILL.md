# Pluribus (skill)

**Pluribus** extends durable memory beyond chat. Use whenever **Pluribus** MCP tools are in your tool list.

## Loop (mandatory)

| Step | Action |
|------|--------|
| 1 | **`recall_context`** — before planning or editing for work that matches **`pluribus-instructions.md`** triggers |
| 2 | **Plan / reason** using the Pluribus recall bundle |
| 3 | **Act** — tools, code, answer |
| 4 | **`record_experience`** — after meaningful outcomes; server decides **at ingest** (probationary memory or reject bucket — no follow-up step) |

Full triggers, doctrine, and fallback when tools are missing: **[`pluribus-instructions.md`](../../../pluribus-instructions.md)** (same text as **`pluribus.mdc`** in this pack).

## WHEN → DO

| WHEN | DO |
|------|-----|
| Non-trivial or multi-step **Pluribus** work | Run step 1 first |
| Durable lesson, decision, failure, or fix | Run step 4 before closing |
| **Pluribus** recall returned little | Still run step 4 when work mattered — empty recall ≠ skip record |

## Forbidden

- Skipping **Pluribus** recall on substantive work when tools are connected  
- Skipping **`record_experience`** after meaningful work  
- Simulating **Pluribus** tool calls you did not make  

## Compatibility

- **`memory_context_resolve`**, **`mcp_episode_ingest`** (legacy **Pluribus** tool names)

## Doctrine

- **Tags + situation** only; no container-shaped memory partitions (**Pluribus** anti-regression / memory-first doctrine)
