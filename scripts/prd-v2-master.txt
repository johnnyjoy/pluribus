--------------------------------------------------
SECTION 1: RECALL INTELLIGENCE ENGINE (RIE)
--------------------------------------------------

PURPOSE
-------
Transform recall from "relevant enough" into "surgically precise".

--------------------------------------------------
1.1 SCORING MODEL
--------------------------------------------------

Each memory object receives a score:

score =
  (authority * 3.0) +
  (recency * 1.5) +
  (scope_match * 2.5) +
  (tag_overlap * 2.0) +
  (failure_proximity * -3.0)

DEFINITIONS:

authority:
  1–10 normalized to 0–1

recency:
  exponential decay:
  recency = exp(-age_days / 30)

scope_match:
  exact match = 1.0
  same project = 0.8
  same domain = 0.5
  global = 0.3

tag_overlap:
  intersection size / union size

failure_proximity:
  if similar to known failure:
  penalty applied

--------------------------------------------------
1.2 RECALL BUCKETS
--------------------------------------------------

Memory must be grouped:

- governing_constraints
- governing_decisions
- known_failures
- applicable_patterns
- contextual_memory

Each bucket has independent limits.

DEFAULT LIMITS:

- constraints: 8
- decisions: 8
- failures: 6
- patterns: 8
- contextual: 10

--------------------------------------------------
1.3 DEDUPLICATION
--------------------------------------------------

Two memories are duplicates if:

- cosine similarity > threshold (future)
OR
- title + tags overlap > threshold (v1)

Duplicates must be merged or highest authority selected.

--------------------------------------------------
1.4 CONTEXT COMPRESSION
--------------------------------------------------

If > N similar memories:

- merge into summary block
- preserve references

Example:

"3 similar failures detected in this subsystem:
- memory_id_1
- memory_id_2
- memory_id_3"

--------------------------------------------------
1.5 RECALL JUSTIFICATION
--------------------------------------------------

Each recalled memory must include:

- score
- why selected
- relevance explanation

--------------------------------------------------
1.6 OUTPUT CONTRACT
--------------------------------------------------

Recall output MUST be:

{
  target,
  task,
  constraints[],
  decisions[],
  failures[],
  patterns[],
  contextual[],
  metadata: {
    total_items,
    token_estimate,
    dropped_items
  }
}

--------------------------------------------------
SECTION 2: MEMORY LIFECYCLE MANAGER
--------------------------------------------------

PURPOSE
-------
Prevent entropy and ensure only high-quality memory survives.

--------------------------------------------------
2.1 STATES
--------------------------------------------------

- candidate
- reviewed
- promoted
- active
- deprecated
- archived

--------------------------------------------------
2.2 AUTHORITY EVOLUTION
--------------------------------------------------

Increase authority if:
- memory used successfully
- referenced repeatedly

Decrease authority if:
- contradicted
- leads to failure

--------------------------------------------------
2.3 SUPERSESSION
--------------------------------------------------

New memory may supersede old:

old.status = deprecated
new.supersedes_id = old.id

--------------------------------------------------
2.4 DECAY
--------------------------------------------------

Low-confidence memory decays:

authority = authority * decay_factor
